Why it's different

Built like a compiler, not eyeballed like a puzzle blog

Nobody manually checks generated boards for correctness — the tests are the referee. That discipline is what lets the engine run offline, instantly, forever.

►

Offline, deterministic generation

Seed in, puzzle out. No server, no network call, same seed always yields the same board.

✓

Guaranteed-unique solutions

Two independently authored solvers plus a validator cross-check every board before it's ever shown.

⋮

Keyboard + mouse

Two-handed keyboard scheme in the TUI, full mouse hit-testing, and point-and-click in the browser.

▦

Five games, one engine

Tango, Queens, Zip, Patches, Mini Sudoku — one pure Go core, compiled to WASM for the web build.

The lineup

Five grid-logic puzzles

Every board is generated fresh, verified valid, and de-duplicated before you ever see it.

☀☽

Tango

Balance suns and moons on a 6×6 grid under equals/cross edge constraints. No three alike in a row.

♛

Queens

Place one queen per row, column, and colored region — with no two queens touching, even diagonally.

└┘

Zip

Draw a single path through every cell in numbered order, respecting walls along the way.

▣

Patches

Tile the grid with numbered rectangular regions so every clue's area matches exactly.

#

Mini Sudoku

Classic Sudoku rules distilled to a smaller, faster grid — every row, column and box, once each.

Or install with Go

Already have a Go toolchain? Install straight from the module path:

# requires Go 1.26+
go install github.com/Jensen95/tui-games/cmd/lig@latest

Then launch it:

lig              # interactive menu
lig games        # list game engines
lig generate --game zip --difficulty hard --seed 42
